## 2. 数字生命体的核心构成

智能体并非单一文件，而是由**接口协议 + 适配层 + 具体算法 + 注册机制**四层组成：

```mermaid
classDiagram
    class AgentCore {
        <<abstract>>
        +name: str
        +reset(observation)
        +select_action(observation) Action
        +learn(feedback, observation)
        +get_visual_state() dict
    }

    class AgentObservation {
        +position
        +grid_size
        +walls
        +rewards
        +human_hint
        +step / episode
    }

    class AgentFeedback {
        +event
        +reward
        +message
        +prev_position / new_position
        +action
    }

    class QLearningAgent {
        +q_table
        +epsilon
        ε-贪心探索 + Q 值更新
    }

    class RuleBasedAgent {
        +collision_memory
        贪心寻奖 + 碰撞记忆
    }

    class env_to_agent_obs {
        <<adapter>>
        沙箱观测 → 智能体观测
    }

    AgentCore <|-- QLearningAgent
    AgentCore <|-- RuleBasedAgent
    AgentCore ..> AgentObservation : 读
    AgentCore ..> AgentFeedback : 学
    env_to_agent_obs ..> AgentObservation : 产出
```

### 可替换性

```mermaid
flowchart LR
    A["继承 AgentCore"] --> B["实现四个生命周期方法"]
    B --> C["在 registry.py 注册"]
    C --> D["运行时 swap_agent() 热切换"]

    style A fill:#1a1f2e,stroke:#4ade80
    style D fill:#1a1f2e,stroke:#f0c040
```

| 方法 | 时机 | 职责 |
|------|------|------|
| `reset` | 每回合开始 | 清空/衰减回合内状态 |
| `select_action` | 每步决策 | 根据观测输出 ↑↓←→ |
| `learn` | 动作执行后 | 用反馈更新内部模型 |
| `get_visual_state` | 每帧广播 | 供前端展示「思维」 |

## 3. 沙箱环境构成

沙箱是智能体感知到的「外部世界」，只暴露**观测**与**反馈**，不暴露内部实现：

```mermaid
graph LR
    subgraph 沙箱状态
        MAP["12×10 网格"]
        WALL["墙体集合"]
        RWD["奖励集合"]
        POS["智能体坐标"]
        HINT["人工指引（可选）"]
    end

    subgraph 对外接口
        RESET["reset()"]
        OBS["get_observation()"]
        STEP["step(action)"]
        HINT_SET["set_human_hint()"]
    end

    MAP --- WALL & RWD & POS & HINT
    RESET --> OBS
    STEP --> OBS
    STEP --> FB["Feedback<br/>event + reward + message"]
```

### 反馈信号（奖励塑形）

```mermaid
flowchart TD
    ACT["执行 Action"] --> CHK{结果判定}

    CHK -->|越界 / 撞墙| C1["event: collision<br/>reward: -1.0"]
    CHK -->|踩到奖励| C2["event: reward / done<br/>reward: +10.0"]
    CHK -->|普通移动| C3["event: step<br/>reward: -0.04"]
    CHK -->|步数 ≥ 200| C4["event: done<br/>reward: -0.5"]

    C1 & C2 & C3 & C4 --> AGT["传入 AgentCore.learn()"]
```

> 智能体**不直接读取地图全局最优路径**，仅通过上述稀疏反馈逐步建立行为策略。

## 4. 单步仿真循环（学习的最小单元）

每一 **tick** 完成一次完整的「感知 → 决策 → 行动 → 学习」闭环：

```mermaid
sequenceDiagram
    participant E as SimulationEngine
    participant S as SandboxEnvironment
    participant A as env_to_agent_obs
    participant G as AgentCore

    E->>S: get_observation()
    S-->>E: Observation
    E->>A: 转换
    A-->>E: AgentObservation

    E->>G: select_action(obs)
    G-->>E: Action (↑↓←→)

    E->>S: step(action)
    S-->>E: new Observation + Feedback

    E->>G: learn(AgentFeedback, new obs)
    E->>E: broadcast 快照 → 前端
```

## 5. 回合（Episode）生命周期

```mermaid
stateDiagram-v2
    [*] --> Idle: 服务启动

    Idle --> Running: start()
    Running --> Tick: 每隔 tick_ms

    state Tick {
        [*] --> Perceive
        Perceive --> Decide
        Decide --> Act
        Act --> Learn
        Learn --> [*]
    }

    Tick --> Running: event ≠ done
    Tick --> EpisodeEnd: event = done

    EpisodeEnd --> Reset: reset_episode()
    Reset --> Running: 新回合

    Running --> Paused: stop()
    Paused --> Running: start()
    Paused --> Tick: step() 单步
```

**回合结束条件：**
- 收集全部奖励
- 达到最大步数（200）

## 6. Q-learning 智能体的学习原理

默认智能体通过 **Q-learning** 在无任何标注轨迹的情况下学习导航：

```mermaid
flowchart TB
    subgraph 决策
        O["当前位置 state"] --> EPS{ε-贪心}
        EPS -->|随机| RND["随机 Action"]
        EPS -->|利用| BEST["argmax Q(state, ·)"]
        HINT["人工指引"] -.->|60% 概率覆盖| EPS
    end

    subgraph 更新
        RND & BEST --> EXEC["执行并得到 reward"]
        EXEC --> BELL["Q(s,a) ← Q(s,a) + α·(r + γ·max Q(s',a') − Q(s,a))"]
        BELL --> QT["Q-table 持久累积"]
    end

    subgraph 衰减
        EP["新回合"] --> DEC["ε ← max(ε_min, ε × decay)"]
    end
```

```mermaid
graph LR
    subgraph 超参数
        α["α = 0.3 学习率"]
        γ["γ = 0.95 折扣因子"]
        ε0["ε = 0.25 → 0.05 探索率"]
    end

    α & γ & ε0 --> QL["QLearningAgent"]
```

## 7. 弱监督：人工指引通道

人工指引不修改 Q 表，而是在**决策阶段**偏置动作选择，属于弱监督：

```mermaid
sequenceDiagram
    actor 用户
    participant UI as 前端 d-pad
    participant E as SimulationEngine
    participant S as SandboxEnvironment
    participant G as QLearningAgent

    用户->>UI: 点击方向键
    UI->>E: hint: up/down/left/right
    E->>S: set_human_hint(Action)
    Note over S: human_hint 写入 Observation

    G->>G: select_action()
    alt random() < 0.6 且 hint 存在
        G-->>G: 直接返回 hint
    else 正常 ε-贪心
        G-->>G: 探索或 argmax Q
    end
```

```mermaid
graph TB
    subgraph 监督强度对比
        FULL["强监督：直接告诉最优路径"] -.->|未采用| X["✗"]
        WEAK["弱监督：偶尔提示方向"] -->|已采用| G["AgentCore.select_action"]
        NONE["无监督：纯环境反馈"] --> G
    end
```
